# Prosper Grand Rules 🦉🦊🐺🐈🧉

このプロジェクト `prosper` は、以下の「グランドルール」を絶対的な憲法として遵守する。
いかなる戦略、実装、運用もこのルールに反してはならない。

## 1. ストレス・フリー (Stress-Free Strategy)
*   **単独完結（No-Humans）**: コミュニケーションコストをゼロにするため、第三者との調整や交渉が発生するフローは採用しない。AIと個人のみで完結させる。
*   **非依存（Independence）**: 特定のプラットフォームやトレンドを追う「他者依存」のルーチンワークは行わない。
*   **非労働型（Passive-Action）**: 毎日コツコツやる作業はNG。初期投資（時間・知能）を最大化し、構築後は「放置」または「監視のみ」の状態を維持する。

## 2. 損失ゼロ (Zero-Loss Strategy)
*   **固定費の極小化**: サーバー代、API利用料、ツール代などの固定費を限りなくゼロに近づける。
*   **時間的損失の排除**: 根拠のない「試し打ち」を禁止する。リサーチ段階で「期待値」を算出し、論理的に勝てると判断したもののみ実行に移す。
*   **不可逆な損失の回避**: アカウント停止リスクや法的リスクを徹底的に精査し、一度の失敗で全てを失うようなリスクは取らない。

## 3. 稼げる (Profit-First Strategy)
*   **必達マネタイズ**: 全ての「アウトプット」には出口（収益化ポイント）を設ける。趣味や実験で終わらせない。
*   **マイクロビジネスの集合体 (The 100x100 Model)**:
    *   大きな一撃（例: 月100万円）を狙うのではなく、月100円の利益を生む「仕組み」を100個構築することを目指す。
    *   これにより、リスクを分散し、一つの仕組みが止まっても全体が崩れない堅牢性を確保する。
*   **PDCAの厳格な順守**: 
    1. **Research**: 徹底した市場調査とロジック構築
    2. **Adjustment**: 仕組みの微調整と検証
    3. **Autonomy**: 仕組み化・自動化
    4. **Operation**: 運用開始
    5. **Result**: 結果の数値化
    6. **Improvement**: 改善（スケールまたは損切り）
- [ ] 収益化パス（キャッシュポイント）の特定
- [ ] 開発環境・販売基盤の構築
    - [ ] GCP/Gemini API連携基盤
    - [ ] 自動デプロイ/販売連携プロトタイプ

## 4. 信頼性と正確性の原則 (Truth & Accuracy)
*   **実証主義**: AIによる「架空の体験談」や「嘘の成功事例」は一切作成しない。
*   **最新性の追求**: コンテンツ作成前には必ず検索ツールを用い、最新のデータ、規約、市場状況を反映させる。
*   **透明性**: AIを活用している場合は、その範囲と価値（効率化、網羅性など）を誠実に開示し、読者を欺かない。
